How DWA-10 Works

Every message runs a deterministic 3-step memory cycle. Priority decays mathematically. Critical facts survive forever.

01 · PRE-CALL
Decay + Pack
All anchor priorities decay exponentially. Top-K anchors selected by utility density and injected into context.
02 · LLM CALL
Anchors + Conversation
Packed memory context prepended to system prompt. LLM receives full context — Claude, GPT-4o, Gemini, any model.
03 · POST-CALL
Extract + Audit
New anchors extracted from response. Chained SHA256 audit event logged. Summary triggered at msg 15 or 70% window.
PRIORITY · P0
Critical — No Decay
Manually flagged facts. Never pruned. Always injected first regardless of token budget.
PRIORITY · P1
Important — Slow Decay
Auto-extracted high-confidence facts. Reinforced on reference. Archived below priority floor.
PRIORITY · P2
Contextual — Fast Decay
Low-confidence extracted facts. Compressed into rolling summary at message 15. Promoted on repeated reference.
